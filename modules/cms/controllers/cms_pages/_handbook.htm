Developer Handbook

The developer handbook contains general information about common API functions. Please read the API section in the LemonStand Wiki in order to learn more about available functions, classes and actions.

Common methods

  • $this->render_partial('partial_name') - renders a partial with specified name.
  • $this->request_param(0, 'default') - returns a URL parameter value by its index.
  • $this->render_page() - outputs a page content. Call this function in page templates in a place where you want the page to be rendered.
  • $this->render_head() - outputs a page head content. Call this function in page templates in a place where you want the page-specific head declarations to be rendered.
  • $this->render_block('sidebar') - outputs a page block defined on the Head & Tabs tab of the page editor. Call this function in page templates in a place where you want to output a specific page block.

Functions

  • content_block('about_us', 'About Us') - outputs a content block with specified code. The second (block name) argument is used in the content editing mode to refer the block. Use only alphanumeric characters for the block code. Please do not use brackets in the block name.
  • global_content_block('footer') - outputs a global content block with specified code. Global content blocks should be created on the CMS/Content page before they can be displayed.
  • checkbox_state(true) - outputs checked='checked' if the parameter value is TRUE. Use it with checkbox INPUT elements.
  • customer_logout('/') - closes a current customer session (logs the customer out) and redirects browser to a specified URL.
  • flash_message() - outputs flash message previously set by a page action.
  • format_currency(99.99) - outputs currency value according currency settings.
  • h('string') - returns an HTML string in safe format, replacing all HTML-sensitive symbols with corresponding HTML sequences, so < becomes &lt;.
  • include_resources() - returns HTML code for including JavaScript libraries required for LemonStand AJAX framework. Call this function in the HEAD element of a page or template.
  • open_form(array('id'=>'my_form')) - outputs an opening FORM element with ACTION attribute value pointing to a current page. The optional array parameter could be used for setting FORM element attribute values.
  • option_state('current', 'selected') - outputs selected="selected" if the first parameter value equals the second one. This function is intended for using with OPTION elements.
  • post('variable', 'default') - returns POST variable value by its name or an optional default value.
  • post_array_item('array', 'variable', 'default') - returns an element from an array posted in the POST request. If the array or element do not exist, returns the default value.
  • radio_state(true) - outputs checked='checked' if the parameter value is TRUE. Use it with radio button INPUT elements.
  • site_url('login') - outputs a full URL: http://www.yoursite.com/login

AJAX requests

LemonStand AJAX framework uses partials and a few lines of JavaScript code. To post AJAX request you should write something like this:

<input type="button" onclick="return $(this).getForm().sendRequest( 'shop:on_addToCart', {update: {'mini_cart': 'shop:mini_cart'} })" value="Add to cart" name="add_to_cart">
